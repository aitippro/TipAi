## 1. 测试总则与通过标准

### 1.1 通过定义（Pass Criteria）

| 级别 | 标准 | 说明 |
|------|------|------|
| **P0-阻塞** | 100% 通过 | 启动失败、崩溃、数据丢失、安全漏洞 |
| **P1-核心** | 100% 通过 | 核心生成流程、AI 调用、数据持久化 |
| **P2-重要** | ≥ 95% 通过 | 业务功能、UI 交互、设置配置 |
| **P3-优化** | ≥ 90% 通过 | UI 美化、动画、边缘场景 |

### 1.2 缺陷分级

| 级别 | 定义 | 响应时间 |
|------|------|----------|
| **Critical** | 应用崩溃、数据丢失、安全漏洞、启动失败 | 立即修复 |
| **Major** | 核心功能不可用、AI 调用失败、数据不一致 | 4 小时内修复 |
| **Minor** | UI 错位、非关键功能异常、体验下降 | 24 小时内修复 |
| **Trivial** | 文案错误、像素偏移、日志噪音 | 下个版本修复 |

### 1.3 测试数据要求

- **每条 AI 调用用例**必须使用真实 API Key 验证，禁止 Mock
- **边界值测试**必须覆盖空字符串、超长字符串（>10KB）、特殊字符（Unicode、Emoji、HTML 标签、SQL 注入片段）
- **并发测试**至少模拟 5 个并行操作
- **长时间运行测试**至少持续 30 分钟
